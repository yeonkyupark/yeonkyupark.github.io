# 2. 빅데이터 분석 계획

## 2.1. 분석 수행 방안 수립

### 2.1.1. 고려사항

1. 전통적인 빅데이터 분석 수행이 가능한 경우 (통계분석, 회귀분석, 추정, 검정)
   1. 주어진 문제에 대하여 구체적으로 정의가 가능
   2. 피룡한 데이터가 존재
   3. 데이터를 분석할 수 있는 역량을 보유
2. 빅데이터 분석 6단계
   1. 문제 인식
   2. 관련 연구조사
   3. 모형화
   4. 자료수집
   5. 자료분석
   6.  분석결과 제시
3. 빅데이터 분석 7단계 (다른 방식)
   1. 수집
   2. 저장
   3. 처리
   4. 분석
   5. 시각화
   6. 이용
   7. 폐기

### 2.1.2. 빅데이터 분석 프로세스
1. 문제 인식
   1. 문제는 가설의 형태로 표현, 분석 과정을 통해 검증
2. 관련 연구조사
  1. 해결하고자 하는 문제를 명확하게 정의, 개괄적으로 어떤 요인(인수)들이 중요하게 문제 해결에 작용하는지 파악
3. 모형화(변수 선정)
   1. 복잡한 현상을 문제의 본질과 관련되는 제어 가능한 변수들로 추려서 단순화는 과정
   2.  `모형화`란 문제를 의도적으로 단순화한 변수들 간의 관계를 정의하는 것
4. 자료수집(변수 측정)
   1. 데이터 자료수집 시 고려사항
     1. 분석 대상 비즈니스의 이해
     2. 데이터 수집 대상
     3. 데이터 위치
     4. 데이터 유형
     5. 수집 방법
     6. 획득비용
  2. 수집된 자료는 데이터베이스에 저장
     1. 데이터베이스 특징 (ACID)
         1. 원자성(Acomicity): 트랜잭션과 관련된 작업들이 부분적으로 실행되다가 중단되지 않는 것을 보장하는 능력
         2. 일관성(Consistency): 트랜잭션 실행 완료 시 언제나 일관성 있는 데이터베이스 상태로 유지
         3. 고립성(Isolation): 트랜잭션 수행 시 다른 트랜잭션의 연산 작업이 끼어들지 못하도록 보장
         4. 지속성(Durability): 성공적으로 수행된 트랜잭션은 영원히 반영
     2. `DBMS(DataBase Management System)`: 다수의 컴퓨터 사용자들이 컴퓨터에 수록된 많은 자료들을 쉽고 빠르게 조회/추가/수정/삭제할 수 있도록 해주는 소프트웨어
     3. 분석 대상에 대한 비즈니스 모델(Business Model), 기업이 수익을 얻기 위한 일련의 행동, 즉 수익모델로 정의
         1. 적합성의 판별 기준: 공공성 < 수익성 
5. 자료분석
    1. 수집 데이터를 분석항려 내재된 의미를 파악
    2. 통계분석부터 데이터마이닝기법에 이르기까지 각가의 상황에 필요한 다양한 데이터 분석기법을 활용
        1. 데이터마이닝: 분류, 예측, 관련성 분석, 군집분석, 소셜네트워크 분석
        2. 소셜네트워크 분석: 노드와 링크로 구성, 네트워크 이론을 기반으로 사람들 사이의 사회적 관계를 분석
6. 분석결과 제시
    1. 변수들 사이의 관련성을 포함한 데이터 분석 결과를 명료하게 해석하여 의사결정자에게 구체적으로 분석 결과를 제시
    2. 표, 그림, 차트 등을 활용하고 전체적으로 문제 정의부터 결과 제시까지 스토리 형식(스토리텔링)을 이용

## 2.2. 분석 절차 및 작업 계획 수립

### 2.2.1. 분석 절차 수립

1. 데이터 분석 절차 비교
   1. 빅데이터 분석 절차 비교

   2. NCS 빅데이터 분석 절차
      1. 도메인 이슈 도출
         1. 문제점 파악을 위해 설문조사를 이용할 경우 `크론바하 알파(Cronbach Alpha)` 값을 이용하여 대상자의 설문 문항 답변에 대한 신뢰도 평가
         2. 최종 결정된 이슈별 개선가ㅗ제, 최종 합의 내용을 토대로 `빅데이터 요건 정의서 작성`
            1. 분석에 대한 기획의도
            2. 빅데이터 분석을 통해 개선되는 부분
            3. 요건 변경 시 기록을 남겨 추적/관리
      2. 분석목표 수립
         1. 목표 수준을 정리, 현실적으로 가능한 분석목표 수립
         2. `분석목표 정의서`
            1. 분석의 기본 정보(분석목적, 우선순위, 접근 방안 등)
            2. 성과측정 방법(정성/정량적 성과측정 기준)
            3. 데이터 관련 정보(내/외부 데이터 정보) 및 분석 타당성에 대한 검토의견(실시간 분석의 가능 여부, 텍스트데이터 분석 수행 여부 등)
      3. 프로젝트 계획 수립
         1. 사전에 책정된 자원(인력 및 인프라), 예산(비용), 기간을 고려
         2. 분석기법, 분석 솔루션, 플랫폼, IT 자원, 데이터 준비상황 등을 점검한 후 WBS 설계
         3. 프로젝터 소요비용(인건비, 하드웨어 및 소프트웨어 비용) 분배, 프로젝트 WBS 작성을 선행
      4. 보유 데이터 자산 확인
         1. 데이터 분석 시작 전 보유 데이터를 점검
         2. 데이터 품질, 분량, 수집경로, 데이터 유형 정리
         3. 데이터별로 컴플라이언스 준수 여부 판단법률적 규칙의 준수 여부 확인
         4. 외부 데이터 수집 시 수집경로 및 피룡한 제약사항을 표기, 개인정보가 포함되었을 경우 제거 또는 대체하여 조직 내부의 법률부서로부터 확인받은 후 사용하고 점검
      5. 빅데이터 분석결과 시각화
         1. 다양한 분석도구 활용, 빅데이터 시각화 수행, 결과 보고서 작성

### 2.2.2. 로드맵 설정과 작업 분할 구조도 설정
1. 프로젝트 수행 로드맵(WBS) 작성 절차
   1. 프로젝트 소요비용 분배
      1. 인건비 배분
      2. 하드에어 비용 배분

      2. 데이터 탐색 및 준비
      3. 데이터 분석 모델링 및 검증
      4. 산출물 및 기타(문서화 등)
   3. 프로젝트 업무 분장 계획 및 배분
      1. 소요기간별 투입인원, 역할정의

      3. 프로젝트 평가(평가위원회 구성)

# 3. 빅데이터 이해

## 3.1. 분석 모형과 부합하는 데이터 정의

### 3.1.1. 빅데이터 서비스 모델
1. 빅데이터 의미
   1. 1990년 이후 인터넷 확산, 정형/비정형 정보가 무수히 발생하면서 정보 홍수나 정보 폭발이라는 개념으로 논의, 최근 `빅데이터`라는 개념으로 이어짐
   2. 모바일 스마트 기기의 확산으로 개인과 관련된 비정형 데이터가 축적되면서 데이터 가중
   3. 빅데이터의 정의는 데이터 규모와 기술 측면에서 출발, 최근 가치와 활용 효과 측면으로 그 의미가 확대
2. 빅데이터의 효과 및 전망
   1. 다변화된 현대 사회를 정확하게 예측하여 효율적인 작동에 기여
   2. 개인화된 현대 사회 구성원마타 맞춤형 정보를 제공/관리/분서 서비스 등을 제공과거 불가능했던 기술을 실현 가능하게 함
   3. 경제, 공공 사회 및 기타 전 영역에서 빅데이터의 도입과 활용은 산업 경쟁력 제고, 생산성 향상, 기능 혁신을 통한 새로운 가치 창출 전망
3. 빅데이터 산업 구조  

    인프라 부분|서비스 부분|&nbsp;|&nbsp;  
    -|-|-|-
    단말/장비 공급자<BR>소프트웨어 공급자|빅데이터 서비스 공급자|데이터 처리<br>데이터 및 정보 제공<br>솔루션 제공<br>교육 및 컨설팅|고객

4. 빅데이터 서비스(비즈니스) 모델
   1. 빅데이터 서비스 제공자(또는 유무선 통신 서비스 제공자)가 단말/장비 공급자와 소프트웨어 공급자로부터 구매한 인프라를 이용하여 고객(사용자)에게 데이터 처리, 데이터 및 정보 제공, 솔루션 제공, 교육 및 컨설팅 제공 등의 시비스를 제공하는 방법을 의미
   2. 데이터 분석 플랫폼 관점에서의 구분
      1. 하드웨어: 빅데이터 저장
      2. 소프트웨어: 데이터 분석
      3. 애플리케이션: 새로운 비즈니스 기회 발굴, 사용자와의 커뮤니케이션, 분석결과의 서비스
   3. 사례
      1. 홈쇼핑 상품추천, 음악 콘텐츠 추천, 신종플루 발병 시기 예측, 소셜네트워크 분석
5. 데이터 수집을 위한 서전 정보
   1. 빅데이터 비즈니스
      1. 비즈니스 모델
      2. 비즈니스 용어
      3. 비즈니스 프로세스
      4. 도메인 전문가 인터뷰
   2. 원천 데이터 정보
      1. 데이터 수집 가능성
      2. 데이터 보안
      3. 데이터 정확성
      4. 수집난이도
      5. 비용

### 3.2.1. 원시 데이터 구성 및 보관 방식

1. 데이터 수집
   1. 데이터 유형
      1. 정형 데이터
         1. 업무처리, 매개거래, 로그 데이터, 시계열 데이터
         2. DB to DB, Sqoop, EAI
         3. ETL 등을 이용한 수집
      2. 반정형 데이터
         1. 데이터 내부에 데이터 구조에 대한 메타 정보 포함
         2. HTML, XML, JSON, RSS, 웹로그, 보안 및 특정 센서 데이터 등
      3. 비정형 데이터
         1. 일반: 소셜 미디어, 고객 서비스, 품질보증, 이벤트, 이메일, 자유형식의 텍스트
         2. 센서 데이터: 온도, QR코드, RFID, GPS
         3. 새로운 데이터 유형: GPS 매칭, 오디오, 정지화상, 비디오
         4. 데이터 수집: Grawler, FTP, HTTP Protocol 수집, Parsing
   2. 내외부 데이터의 주요 특징 및 예
      1. 내부 데이터
         1. 내부 조직간 협의를 통한 데이터 수집
         2. 수집이 용이, 정형 데이터
         3. 비용 및 난이도 용이
         4. 서비스의 수명 주기 관리 용이
         5. 서비스 시스템(ERP, CRM 등), 시스템/서버 장비 로그, 마케팅 자료(VOC, 고객 포털 시스템 등)
      2. 외부 데이터
         1. 외부 조직과 협의, 데이터 구매, 휍상의 오픈 데이터
         2. 수집이 어려운 비정형 데이터
         3. 비용 및 난이도 높음
         4. 외부 환경에 대한 통제가 어려움에 따라 서비스 관리정책 요구
         5. 소셜데이터, 정책 데이터, 토론 사이트, 센서 데이터, 장비간 발생 로그, 공공 데이터 등
   3. 데이터 수집 관련 내용
      1. 데이터 유형: 정형 / 반정형 / 비정형
      2. 데이터 위치: 내부 / 외부
      3. 데이터 저장: 파일 / 관계형 DB / 분산처리
      4. 수집 기술: 정형 데이터 / 로그 데이터 / 웹크롤링 / SNS 데이터
      5. 수집 비용: 종류 / 크기 및 보관 주기 / 수집 주기 / 수집 방식 / 수집 기술 / 데이터 가치
   4. 데이터 수집 방법
      1. 웹상에서의 주요 데이터: HTML, XML, JSON
      2. 데이터 수집 프로그램: Python, Ruby
   5. `웹마이닝(Web Mining)`: 웹페이지로부터 데이터를 수집하고 분석하는 방법
      1. 정의: 인터넷을 이용한 웹서비스의 다양한 패턴(특징)을 발견하기 위해 사용되는 기술로 웹로그 분석, 웹콘텐츠 마이닝, 웹구조 마이닝 등이 해당
   6. 데이터 품질 점검 항목
      1. 데이터 분량: 테이블 내 필요 칼럼별 확인, 칼럼별 데이터 축적 기간 및 분량
      2. 데이터 완전성: 데이터 내 필요한 대상과 속성을 포함하는지 확인, 데이터 누락 또는 결측값의 비율 확인
      3. 데이터 일관성: 데이터 속성간 관계, 데이터 상위/하위 간 관계에서의 값 일치, 데이터 유형과 값의 일치
      4. 데이터 정확성: 데이터의 편향(Bias)와 분산
   7. 데이터 수집 시 고려사항

2. 데이터 저장  
   1. 데이터 유형별 저장 시스템

      유형|데이터 종류|저장 시스템
      -|-|-
      정형|RDB, 스프레드시트|RDB
      반정형|HTML, XML, JSON, 웹문서, 웹로그, 보안 및 특정 센서 데이터|RDB, NoSQL
      비정형|소셜 데이터, 문서(워드, 한글), 이미지, 오디오, 비디오, IoT|NoSQL, 분산파일시스템(HDFS)
      
      1. `데이터웨어하우스(Data Warehouse)`, 기업의 의사결정 과정을 지원하기 위한 주제 중심적이고 통합적이며, 시간성을 가지는 비휘발성 자료의 집합
      2. `데이터마트(Data Mart)`, 전사적으로 구축된 데이터웨어하우스로부터 특정 주제, 부서 중식믕로 구축된 소규모 단일 주제의 데이터웨어하우스(저장소)
      3. `EAI(Enterprise Appllication Integration)`, 기업 내 또는 기업들 사이의 이질적인 슷템을 효율적으로 연계하여 메시지를 통합처리하는 기술
   2. 데이터 저장 방법
      1. 분산파일시스템
         1. 확장 가능한 분산파일 형태로 저장
         2. 대용량, 분산, 데이터 집중형의 애플리케이션 지원
         3. 고성능 Fault-toerance 환경 제공
         4. Apache HDFS(Hadoop Distributed System)
         5. Google GFS(Google File System)
      2. 데이터베이스(NoSQL)
         1. 데이터베이스에 대한 기본 개념 활용
         2. 파일관리에서부터 시작하여 편리성을 도모하기 위하여 데이터베이스로 발전
         3. RDBMS와 NoSQL로 구분
         4. 관계형 데이터베이스 시스템: RDBMS
         5. NoSQL 데이터베이스 시스템
         6. Key-value DB
            1. 키와 해당 키 값의 쌍으로 저장하는 데이터 모델
            2. 아마존 Dynamo, Redis 오픈소스 데이터베이스
         7. Column(또는 Row) oriented DB
         8. Column(또는 Row)을 여러 노드로 분할하여 저장
         9. 데이터를 열(또는 행) 기반으로 저장, 확장성 보장
         10. 구글 빅테이블, Cassandra, HBase, HyperTable
     1. Document DB
        1. 문서 형식의 데이터 저장
        2. 복잡한 형태의 데이터 저장 지원 및 최적화
        3. MongoDB, SimpleDB, CouchDB
     2. NoSQL 및 MongoDB 시스템 특징
        1. NoSQL
           1. 빅데이터와 클라우드 컴퓨팅의 도입으로 기존 RDBMS 한계점 극복
           2. 분산 구조와 Fault-tolerant 구조 지원, 고정된 Data Schema가 아님
           3. 비싼 비용의 join 연산이 없음
           4. RDBMS의 ACID(원자성, 일관성, 고립성, 지속성) 특징 중 하나를 완화함
           5. CAP(Consistency, Avaliability, Partition) 정리의 성질 만족
           6. 분산 환경에서 가용성 지원
           7. 저비용으로 쉽게 분산 환경의 대량 데이터 저장 시스템 구축
           8. 여러 노드(서버)에 데이터 복제, 저장(단편화)
           9. Consistency 제약 조건을 일부 완화하여 시스템 구축
        2. MongoDB
           1. 오픈소스 DBMS, 문서 기반 데이터베이스의 쉬운 개발과 확장성 확보
           2. 높은 성능: 데이터 입출력 시간을 줄이기 위해 Join이 필요없는 Embedded Data Model 지원
           3. 빠른 질의 처리를 위한 인덱스 지원, Embedded 문서와 배열에 있는 key 값을 indexing할 수 있음
           4. 높은 가용성: 분산 시스템 환경에서 자동 Failover 및 데이터 중복 지원
           5. 자동 확장성: 자동 Sharding(데이터 분산 저장 기법) 지원
           6. 중복 데이터 집합에 대한 일관성 읽기 지원
           7. `샤딩(Sharding)`, 데이터 분산 저장을 위한 기법,대용량 데이터베이스에서 데이터를 수평분할하는 방법, RDBMS의 파티셔닝 개념과 일치
     3. 데이터 분산 저장 및 처리
        1. 분산 컴퓨팅
           1. 단일 시스템의 한계 극복
           2. 단일 시스템의 성능 향상을 위한 비용증가로 효율성 감소
           3. 네트워크로 연결된 시스템에 여러 장치를 분산하여 처리
           4. 대형 시스템의 복잡성을 줄이고 다양한 보안 정책이 가능
           5. 각 시스템의 개별적 독립성 제공
        2. 병렬 검퓨팅
           1. 여러 개의 복잡한 연산을 순차적이 아닌 병렬적으로 동시 처리
           2. 서로 독립적 결과를 얻는 병렬적 처리 단계로 변환하여 문제 해결
           3. 동시에 수행되어도 상관없는 처리 단계들로 구성
           4. 어떤 단계의 결과가 다른 단계에 영향을 미치지 않음
           5. 병렬화: 동시에 처리되는 대상을 찾아 구분하는 작업
           6. 병렬화가 가능한 수준만틈 처리시간 단축 가능
           7. 특정 데이터가 다른 데-이터에 영향을 주지 않는 경우 적용
           8. 데이터는 독립적으로 저장, 처리, 분석될 수 있음
           9. 병렬 컴퓨팅을 통해 한 번에 많은 데이터 처리 가능
        3. 클라우드 구조 
            1. 인터넷(웹) 기반의 컴퓨팅 기술
            2. 유틸리티 데이터 서버에 프르그램을 두고 필요 시 컴퓨터(단말기) 등에 불러와서 사용
            3. 인터넷 IT 자원(소프트웨어, 플랫폼, 인프라 등) 사용
            4. IT 자원의 소유가 아니라 대여의 개념
            5. 보다 많은 IT 자원을 이용할 수 있는 서비스 제공
            6. 문서 작성 및 저장장치를 통한 데이터 저장 기능
            7. IT 서비스로써 소프트웨어, 인프라, 플랫폼의 개념으로 발전
      3. 분산 검퓨팅 하드웨어 구조
         1. 리드 컴퓨팅
         2. GPGPU(Genernal Purpose Computing on Graphics Processing Unit)
